# Contributing to Proddit

Thank you for your interest in contributing to Proddit! We welcome contributions of all kinds, from bug fixes to new features.

To ensure a smooth collaboration, please follow these guidelines:

## General Principles

1.  **Preserve Architecture**: Please do not make significant changes to the existing file structure or the core logic unless explicitly discussed and approved. We aim to keep the codebase consistent and easy to navigate.
2.  **Ask for Help**: If you get stuck or are unsure how a particular part of the system works, don't hesitate to ask questions! We're here to help.
3.  **Atomic Changes**: Keep your pull requests focused on a single issue or feature. Smaller, atomic PRs are much easier to review and merge.

## How to Contribute

### Reporting Issues
- If you find a bug or have a suggestion for improvement, please report it by creating an issue.
- Provide as much detail as possible, including steps to reproduce bugs.

### Feature Additions & Enhancements
- We welcome new features and enhancement ideas!
- If you have a new idea, feel free to open an issue to discuss it before starting work.

### Submitting Pull Requests
1.  **Fork the Repository**: Create a fork and work on a new branch.
2.  **Follow Coding Style**: Try to match the existing coding style and conventions used in the project.
3.  **Write Clear Descriptions**: When submitting a PR, provide a clear summary of what was changed and why.
4.  **Verify Your Changes**: Ensure your code runs correctly and doesn't break existing functionality before submitting.

## Communication
- Be respectful and constructive in all interactions.
- We value your time and effort – thank you for helping make Proddit better!
